# Plan: Implement Vote-Based NDAU Signing in NdauConversionPage

## Problem

The NDAU wallet doesn't support custom message signatures. The current implementation uses `useNdauSigner` which tries to sign custom messages via `website-sign-request-server` event, but this fails silently. The wallet only supports vote-style signing.

## Solution

Replace the custom message signing with the "Sign Payload API" documented in NDAU_WALLET_INTEGRATION.md, which sends the signing request as if it were a vote.

## Changes Required

### 1. Check/Create Account Endpoint

- Verify if `/api/account` endpoint exists in server to get validation key
- If not, this needs to be created or use a different approach

### 2. Modify NdauConversionPage.tsx

- Replace `useNdauSigner` hook usage with vote-based signing flow
- Create YAML payload with vote structure:
  ```yaml
  vote: yes
  proposal:
    proposal_id: ndau-to-revo-conversion
    proposal_heading: "I agree to convert my ndau to the Ethereum address: {revoAddress}"
    voting_option_id: 1
    voting_option_heading: "Confirm Conversion"
  wallet_address: { ndauAddress }
  validation_key: { validationKey }
  ```
- Convert payload to base64: `btoa(yaml.stringify(payload))`
- Send POST to `/api/sign` with payload, walletAddress, websiteSocketId
- Listen for WebSocket events:
  - `server-sign-fulfilled-website` → signature received
  - `server-sign-rejected-website` → user rejected
  - `server-sign-failed-website` → error occurred

### 3. Dependencies

- Check if `yaml` package is available, or use JSON instead
- Need access to WebSocket from NdauWalletContext (already available)

## Implementation Order

1. First verify the account/validation key endpoint exists
2. Update handleSignNdau function in NdauConversionPage.tsx
3. Add WebSocket listeners for new events
4. Test the flow
