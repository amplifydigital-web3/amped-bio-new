# Plan: Remover ndauAmount do submitConversion (Fetch pelo Backend)

## Problema

O `ndauAmount` está sendo enviado do front-end para o backend no `submitConversion`. Isso é um problema de segurança pois o front-end pode manipular o valor. O backend deve buscar esse valor automaticamente.

## Solução

O backend já tem a função `getNdauBalance` que busca o balance via API externa. Vamos reutilizar essa lógica no `submitConversion`.

---

## 修改s Necessárias

### 1. Backend: `apps/server/src/trpc/ndauConversion.ts`

**Input do submitConversion:**

- Remover `ndauAmount` do input schema
- Manter apenas: `ndauAddress`, `revoAddress`, `ampedbioSignature`, `ndauSignature`, `ndauValidationKey`

**Mutation submitConversion:**

- Dentro do mutation, apos obter o `ndauAddress`:
  1. Fazer fetch do balance usando a mesma lógica do `getNdauBalance`
  2. Obter o `ndauAmount` da API
  3. Validar que o balance é > 0
  4. Usar esse valor para calcular `revoAmount` e salvar no banco

### 2. Frontend: `apps/client/src/pages/NdauConversionPage.tsx`

- Remover `ndauAmount` do objeto enviado no `submitMutation.mutate()`
- Manter apenas: `ndauAddress`, `revoAddress`, `ampedbioSignature`, `ndauSignature`, `ndauValidationKey`
- Validacao local de `ndauBalance` pode ser mantida para UX (avisar usuario antes), mas o valor oficial vem do backend

---

## Riscos/Consideracoes

1. **API externa**: Se a API do NDAU estiver indisponível, o submit pode falhar. Isso é esperado e deve retornar erro claro.
2. ** Timing**: O balance pode mudar entre a exibicao no front e o submit. O_backend pegará o valor atual, o que é mais seguro.
3. ** Validação**: O backend deve verificar se o balance é suficiente para conversao.

---

## Files Afetados

- `apps/server/src/trpc/ndauConversion.ts` - Input e mutation
- `apps/client/src/pages/NdauConversionPage.tsx` - Submit mutation call
